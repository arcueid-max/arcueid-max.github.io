通用问题

RAG 和普通大模型问答有什么区别？

普通大模型直接基于已有参数生成答案，可能会出现“幻觉”。 RAG 会先从知识库检索相关内容，再结合模型生成，保证答案更加准确、可控、可溯源。

为什么需要向量数据库？

向量数据库用于存储文档的语义向量表示。 在检索时，它可以快速找到与用户问题最相似的内容，比传统关键词搜索更智能。

如何防止大模型生成错误内容？

通过 RAG 将外部知识库作为“证据”，模型在回答时必须参考文档，从而减少幻觉。 同时可以在输出端增加“引用来源”，提升可信度。

技术相关

前端为什么用 GitHub Pages？

GitHub Pages 免费、方便部署，适合展示静态网站。 它只负责展示页面，实际的大模型推理和检索在后端完成。

后端可以用哪些平台部署？

- Vercel（适合轻量级 API）
- Hugging Face Spaces（支持 ML 推理）
- 阿里云 / 腾讯云 / AWS（适合生产环境）

如何扩展知识库？

只需要将新的资料进行向量化，并写入向量数据库。 模型调用时就会自动检索到新资料，不用重新训练。

项目相关

我们的大创项目主题是什么？

项目主题是答而有据 - 基于大模型知识增强的智能问答学习平台， 目标是为学习和科研提供一个有来源支撑的智能问答助手。

网站上线后，用户能直接用大模型吗？

用户可以在网站输入问题，前端会调用我们后端的 RAG API， 返回的答案带有知识来源，保证更可靠。
