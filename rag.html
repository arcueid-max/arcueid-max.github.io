什么是 RAG？

RAG（Retrieval-Augmented Generation） 是一种结合了 信息检索 (Retrieval) 与 文本生成 (Generation) 的方法。 它的核心思想是：在大模型生成回答之前，先从知识库中检索相关内容， 然后将这些检索到的内容作为上下文提供给大模型，使回答更加准确、可控、可解释。

RAG 工作流程

  1. 用户输入问题 → 系统接收自然语言问题。
  2. 检索模块 → 从向量数据库中找到最相关的文档。
  3. 生成模块 → 将问题 + 检索到的文档交给大语言模型 (LLM)。
  4. 输出答案 → 模型基于上下文生成更准确的回答。

常见应用场景

  • 📖 智能学习助手（结合教材/论文）
  • 🛠️ 企业知识库问答
  • ⚡ 技术支持机器人
  • 🔍 法律、医疗等专业领域问答

代码示例 (Python)


from langchain.chains import RetrievalQA
from langchain.chat_models import ChatOpenAI

# 加载大语言模型
llm = ChatOpenAI(model="gpt-3.5-turbo")

# 创建 RAG 问答链
qa = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=向量数据库.as_retriever()
)

# 提问
result = qa.run("RAG 的工作流程是什么？")
print(result)
